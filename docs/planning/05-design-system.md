# Design System (기초 디자인 시스템)

> 비기술자도 이해할 수 있는 디자인 언어로 작성합니다.
> 최소 접근성 패키지를 포함합니다.

## 1. 디자인 철학

### 1.1 핵심 가치

| 가치 | 설명 | 구현 방법 |
|------|------|----------|
| **단순함** | 복잡한 UI를 피하고 1~2번 터치로 완료 | 큰 버튼, 명확한 레이블, 불필요한 메뉴 제거 |
| **친근함** | 따뜻하고 재미있는 느낌 | 파스텔톤 색상, 이모지 활용, 격려하는 메시지 |
| **신뢰감** | 안정적이고 전문적인 느낌 | 일관된 디자인, 명확한 피드백, 에러 처리 |

### 1.2 참고 서비스 (무드보드)

| 서비스 | 참고할 점 | 참고하지 않을 점 |
|--------|----------|-----------------|
| **Planning Center** | 현대적이고 부드러운 파스텔톤 디자인, 직관적인 UI | 과한 일러스트, 복잡한 애니메이션 |
| **Breeze ChMS** | 위젯 기반 대시보드, 커스텀 필드 | 모노톤 컬러, 딱딱한 느낌 |
| **디모데** | 계층적 조직 구조, 심방 디테일 | 구식 UI, 번역체 느낌 |

## 5. 기본 컴포넌트

### 5.1 버튼 (Button)

| 상태 | Primary | Secondary | Ghost |
|------|---------|-----------|-------|
| 기본 | 파스텔 블루 배경 + 흰 텍스트 | 테두리 + 파스텔 블루 텍스트 | 텍스트만 |
| 호버 | 약간 어둡게 (`#2563EB`) | 배경 채움 | 밑줄 |
| 포커스 | 포커스 링 (2px, 파스텔 블루) | 포커스 링 | 포커스 링 |
| 비활성 | 50% 투명도 | 50% 투명도 | 50% 투명도 |
| 로딩 | 스피너 표시 | 스피너 표시 | 스피너 표시 |

**크기:**
- Large: 높이 48px, 패딩 24px (모바일 터치 최적화)
- Medium: 높이 40px, 패딩 16px (기본)
- Small: 높이 32px, 패딩 12px

**예시:**
```tsx
<Button size="lg" variant="default">
  출석 체크하기
</Button>
```

### 5.2 입력 필드 (Input)

| 상태 | 스타일 |
|------|--------|
| 기본 | 테두리 (`#E5E7EB`) + 배경 (`#FFFFFF`) |
| 포커스 | 파스텔 블루 테두리 (`#3B82F6`) + 포커스 링 |
| 에러 | 빨강 테두리 (`#EF4444`) + 에러 메시지 표시 |
| 비활성 | 회색 배경 (`#F3F4F6`), 입력 불가 |

**예시:**
```tsx
<Input
  type="email"
  placeholder="이메일을 입력하세요"
  error="올바른 이메일 형식이 아닙니다"
/>
```

### 5.3 카드 (Card)

- 배경: 흰색 (`#FFFFFF`)
- 테두리: 1px solid (`#E5E7EB`)
- 모서리: 8px 라운드 (`rounded-lg`)
- 그림자: `shadow-sm` (0 1px 2px rgba(0,0,0,0.05))
- 내부 여백: 16px (`p-4`)

**모바일 최적화:**
- 모든 테이블은 모바일에서 카드 형태로 변환
- 학생 리스트는 카드 그리드로 표시

**예시:**
```tsx
<Card>
  <CardHeader>
    <CardTitle>김철수</CardTitle>
    <CardDescription>3학년, 사랑반</CardDescription>
  </CardHeader>
  <CardContent>
    <Button>출석 체크</Button>
  </CardContent>
</Card>
```

### 5.4 배지 (Badge)

| 타입 | 색상 | 사용처 |
|------|------|--------|
| Success | 초록 배경 + 흰 텍스트 | 출석 완료 |
| Warning | 노랑 배경 + 흰 텍스트 | 경고 |
| Error | 빨강 배경 + 흰 텍스트 | 결석 |
| Info | 파랑 배경 + 흰 텍스트 | 정보 |

## 6. 접근성 체크리스트

### 6.1 필수 (MVP)

- [x] **색상 대비**: 텍스트와 배경의 대비율 4.5:1 이상
  - Primary 텍스트 (`#111827`) on Background (`#F9FAFB`) = 15.8:1 ✅
  - Secondary 텍스트 (`#6B7280`) on Background = 4.6:1 ✅
- [x] **포커스 링**: 키보드 탐색 시 포커스 표시 명확 (2px, 파스텔 블루)
- [x] **클릭 영역**: 버튼/링크 최소 44x44px (모바일 터치 최적화)
- [x] **에러 표시**: 색상만으로 구분하지 않음 (아이콘/텍스트 병행)
- [x] **폰트 크기**: 본문 최소 14px (16px 사용)

### 6.2 권장 (v2)

- [ ] 키보드 전체 탐색 가능
- [ ] 스크린 리더 호환 (ARIA 라벨)
- [ ] 애니메이션 줄이기 옵션
- [ ] 고대비 모드

## 7. 아이콘 & 일러스트

### 7.1 아이콘 라이브러리

| 옵션 | 설명 | 링크 |
|------|------|------|
| **Lucide** | 깔끔한 라인 아이콘, Shadcn/UI 기본 | lucide.dev |
| Heroicons | Tailwind 공식 (대안) | heroicons.com |

**선택: Lucide**
- Shadcn/UI와 통합되어 있음
- 한글과 잘 어울리는 깔끔한 디자인

### 7.2 아이콘 사용 규칙

- 크기: 16px (작음), 20px (기본), 24px (큼)
- 색상: 텍스트와 동일하게 상속
- 버튼 내: 텍스트 왼쪽에 배치, 8px 간격

**예시:**
```tsx
<Button>
  <CheckCircle className="w-5 h-5 mr-2" />
  출석 체크
</Button>
```

## 8. 톤앤매너 (Tone & Manner)

### 8.1 말투

**재미있고 활기찬 느낌 (게이미피케이션)**

**예시:**
- ✅ "오늘 출석 체크 완료! 🎉"
- ✅ "와! 오늘 출석률 100%!"
- ✅ "김철수 학생이 오늘 출석했습니다 🙏"
- ✅ "3주째 결석 중입니다. 심방이 필요해요!"

**금지 표현:**
- ❌ "출석 처리가 완료되었습니다." (딱딱함)
- ❌ "Attendance has been recorded." (번역체)
- ❌ "오류가 발생했습니다." (부정적)

### 8.2 이모지 사용

**적절한 사용:**
- 성공/완료: 🎉, ✅, 🎊
- 경고: ⚠️, 🔔
- 정보: ℹ️, 💡
- 격려: 🙏, 💪

**과도한 사용 금지:**
- 한 문장에 이모지 2개 이상 사용하지 않음
- 공식적인 메시지에는 이모지 제한

## 9. 모바일 최적화

### 9.1 반응형 브레이크포인트

| 브레이크포인트 | 크기 | 용도 |
|---------------|------|------|
| Mobile | < 640px | 스마트폰 (기본) |
| Tablet | 640px ~ 1024px | 태블릿 |
| Desktop | > 1024px | 데스크톱 |

### 9.2 모바일 퍼스트 원칙

- 모든 디자인은 모바일부터 시작
- 테이블은 모바일에서 카드 형태로 변환
- 터치 영역 최소 44x44px
- 스와이프 제스처 활용 (v2)
