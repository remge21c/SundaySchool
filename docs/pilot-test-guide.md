## 테스트 진행

### 테스트 기간
- **권장 기간**: 1-2주
- **최소 기간**: 1주 (주일 1회 출석 체크 포함)

### 테스트 시나리오

#### 시나리오 1: 주일 출석 체크 (필수)
1. **목표**: 매주 주일 예배 전/후 출석 체크
2. **절차**:
   - 로그인
   - 출석 체크 페이지 접속
   - 반 선택
   - 각 학생의 출석 상태 체크
   - 결석 시 사유 입력
3. **예상 소요 시간**: 3-5분
4. **성공 기준**: 
   - [ ] 모든 학생의 출석 상태가 정확히 기록됨
   - [ ] 출석 체크 완료 시간이 3분 이내임
   - [ ] 시스템 오류 없이 정상 작동함

#### 시나리오 2: 학생 정보 확인 (권장)
1. **목표**: 학생 프로필 정보 확인 및 수정
2. **절차**:
   - 대시보드에서 학생 목록 확인
   - 특정 학생 프로필 열기
   - 학생 정보 확인 (이름, 생년월일, 연락처 등)
   - 필요 시 정보 수정
3. **예상 소요 시간**: 5-10분
4. **성공 기준**:
   - [ ] 학생 정보가 정확하게 표시됨
   - [ ] 정보 수정이 정상적으로 저장됨

#### 시나리오 3: 심방 기록 (권장)
1. **목표**: 학생 심방 후 기록 작성
2. **절차**:
   - 학생 프로필 페이지 접속
   - "심방 기록" 탭 클릭
   - 심방 기록 작성 (유형, 내용, 기도제목)
   - 저장
3. **예상 소요 시간**: 5-10분
4. **성공 기준**:
   - [ ] 심방 기록이 정상적으로 저장됨
   - [ ] 타임라인에서 기록 확인 가능

#### 시나리오 4: 장기 결석 알림 확인 (선택)
1. **목표**: 장기 결석 학생 파악
2. **절차**:
   - 대시보드에서 "장기 결석 알림" 확인
   - 3주 이상 결석한 학생 목록 확인
   - 필요 시 심방 계획 수립
3. **예상 소요 시간**: 2-3분
4. **성공 기준**:
   - [ ] 장기 결석 학생 목록이 정확하게 표시됨

## 문제 해결

### 자주 발생하는 문제

#### 문제 1: 로그인이 안 되는 경우
**증상**: 이메일/비밀번호를 입력해도 로그인되지 않음

**해결 방법**:
1. 이메일/비밀번호 오타 확인
2. Supabase 대시보드 → Authentication → Users에서 계정 존재 여부 확인
3. 브라우저 개발자 도구(F12) → Console 탭에서 에러 확인
4. 문제가 계속되면 관리자에게 문의

#### 문제 2: 학생 정보가 보이지 않는 경우
**증상**: 로그인은 되지만 학생 목록이 비어있음

**해결 방법**:
1. 관리자 페이지에서 담당 반 배정이 되어 있는지 확인
2. 담당 반에 학생이 등록되어 있는지 확인
3. 문제가 계속되면 관리자에게 문의

#### 문제 3: 출석 체크가 저장되지 않는 경우
**증상**: 출석 상태를 변경했지만 저장되지 않음

**해결 방법**:
1. 네트워크 연결 확인
2. 페이지 새로고침 후 다시 시도
3. 브라우저 개발자 도구(F12) → Console 탭에서 에러 확인
4. 문제가 계속되면 관리자에게 문의
